# NEXAH Dynamical Systems Toolbox

The `tools` module contains analysis utilities for exploring dynamical structures within NEXAH regime landscapes.

While the NEXAH kernel itself focuses on **navigation of structural systems**, the tools in this directory allow deeper investigation of the **dynamical properties of those landscapes**.

These tools are primarily intended for:

- dynamical systems exploration
- resonance detection
- attractor analysis
- chaos diagnostics
- parameter space structure analysis

Most tools operate on data generated by kernel simulations and scans located in:

```
ENGINE/nexah_kernel/demos/data
```

Visual outputs are typically written to:

```
ENGINE/nexah_kernel/demos/visuals
```

---

# Conceptual Role

The toolbox extends the NEXAH kernel into a **dynamical systems exploration environment**.

While the kernel answers:

> How can an agent navigate a structural system?

The toolbox investigates:

> What is the dynamical structure of the system landscape itself?

This includes identifying:

- attractors
- resonance zones
- quasiperiodic regions
- chaotic regimes
- fractal parameter structures

# Core Analysis Modules

## Resonance Discovery

| Tool | Description |
|-----|-------------|
| discover_resonance_zones.py | Identifies high-scoring resonance regions |
| resonance_ridge_detector.py | Detects ridge structures in resonance landscapes |
| resonance_harmonic_analyzer.py | Extracts harmonic spectra of resonance zones |
| resonance_topology_map.py | Builds topological maps of resonance landscapes |

These tools identify structural resonance patterns in parameter space.

---

## Attractor & Stability Analysis

| Tool | Description |
|-----|-------------|
| nexah_resonance_attractor_finder.py | Detects attractors in resonance landscapes |
| nexah_lyapunov_map.py | Computes Lyapunov stability across parameter space |
| nexah_kam_torus_detector.py | Identifies quasi-periodic invariant torus regions |
| nexah_kam_surface_plot.py | Visualizes KAM stability surfaces |

These tools help distinguish stable, chaotic, and quasiperiodic dynamics.

---

## Frequency Locking & Circle Map Diagnostics

| Tool | Description |
|-----|-------------|
| nexah_rotation_number_analysis.py | Computes rotation numbers across parameter space |
| nexah_arnold_tongue_map.py | Detects frequency-locking Arnold tongues |
| nexah_devils_staircase.py | Generates devil's staircase plots |

These diagnostics are typical for **circle map and forced oscillator systems**.

---

## Fractal Structure Analysis

| Tool | Description |
|-----|-------------|

| nexah_fractal_dimension.py | Estimates fractal dimension of resonance regions |

## Universality & Critical Phenomena

| Tool | Description |
|-----|-------------|

# Typical Workflow

A typical analysis pipeline may look like this:

1️⃣ Generate parameter scans

```
resonance_scan_full.json
```

2️⃣ Detect resonance zones

```
python -m ENGINE.nexah_kernel.tools.discover_resonance_zones
```

3️⃣ Analyze dynamical properties

```
python -m ENGINE.nexah_kernel.tools.nexah_rotation_number_analysis
python -m ENGINE.nexah_kernel.tools.nexah_lyapunov_map
```

4️⃣ Detect structural phenomena

```
python -m ENGINE.nexah_kernel.tools.nexah_kam_torus_detector
python -m ENGINE.nexah_kernel.tools.nexah_arnold_tongue_map
```

5️⃣ Explore universality properties

```
python -m ENGINE.nexah_kernel.tools.nexah_feigenbaum_analysis
```

# Relation to the Kernel

The tools are **not required for the kernel to function**.

Instead, they act as a **research layer** on top of the kernel.

Kernel:

```
system → regime landscape → navigation
```

Toolbox:

```
regime landscape → dynamical structure → universality
```
